## 3. このセッション（S2）で入った修正の全リスト

### 第1ラウンド（HANDOVER_20260611 の承認済みスコープ A/B/C）
**A. 挙動系**
1. **M-8 yaw規約統一**: converter は Apollo 規約（d 系は空間微分）のまま、`frenet_planner._cartesian_to_frenet_state` で時間微分へ変換（`ḋ=d'·ṡ`, `d̈=d''·ṡ²+d'·s̈`）、`_calc_global_paths` で逆変換（`d'=ḋ/ṡ`、`EPS_S_DOT=1e-3` 未満は d'=d''=0 フォールバック）。回帰テスト `tests/test_frenet_conventions.py`（fp.yaw vs ポリライン接線 5°以内、旧実装は最大27°乖離）。
2. **M-7 横格子対称化**: `n_side = int(w/dw+1e-9); di_values = arange(-n_side, n_side+1)*dw` — d=0 必含・完全対称。
3. **CAUTION 速度制約 index 0 除外**（→ 第2ラウンドで加速度にも拡張）。
4. **ホライゾン端点包含**: `_build_time_cache` が t=Ti を含む格子、Ti レンジが max_t を含む。予測51点配列（前置込み）と index がちょうど整合（clip 既存）。
5. **スプライン端切り詰めのロックステップ化**: 引き継ぎ書の文言「len(x)!=len(t) を invalid」は **F1 致命バグの再導入になるため不採用**。代わりに Cartesian + Frenet 全15配列を `[:first_nan_idx]` で同時切り詰め（`first_nan_idx<2` は全配列空化=無効）。`_check_paths` 冒頭に `len(x)!=len(t)` 防御ガード。
6. **エゴ曲率キャッシュ**: `_last_kappa`（前回採用経路の c[1]、シミュレータは index 1 へ進むため）。
7. **M-10 復帰閾値**: デフォルト caution 0.5→**2.0** / emergency 1.0→**3.0**（中心間）。`validate_config` で結合半径超を強制（multi_circle は footprint 半径基準）。旧デフォルトは clearance 換算 −0.7/−0.2m で距離ゲートが無効だった。
