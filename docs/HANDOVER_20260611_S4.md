## 0. 新ミッション（ユーザー指示、優先順）

1. **近接表示の解明と修正**: S3 アニメーション（t=10.6s）で車両矩形と歩行者が接触寸前に見える。
   - **調査済みの事実**: これは描画の誇張ではない。animator は車体 4.5×2.0m をハードコードで正直に描画（`src/visualization/animator.py:354-355`）。一方、circle モードの衝突判定は中心の r=1.0 円（衝突閾値 = 中心間 1.2m）。中心間 1.3〜1.4m の歩行者は**実際に矩形と重なっている**（FINAL_BENCHMARK_REPORT.md §4 の「論文構成 50/123 ランが矩形違反」と同根）。
   - 対応案: (a) 可視化に「実際に使われている衝突 footprint」（circle: r=1.0 円 / multi_circle: 3円 r=1.25）をオーバーレイ描画し、矩形との乖離を見えるようにする（誠実な可視化、低コスト）。(b) 根本対策は multi_circle 化（Gen-5b の E、§3 参照）。(c) ついでに animator のハードコード L/W を config（vehicle_length/width）から取るよう修正。
2. **急ブレーキの根絶（緩やかで安全な減速）**: 速度プロファイルは「5 m/s から先行減速 → 2 m/s 付近から −4 m/s² で急停止」という形が残っている。
   - **構造的原因（特定済み）**: ① プランナの停止プロファイルは最短でも Ti=min_t=4.0s（`min_t` config）で、4〜5秒かけた緩停止しか生成できない。衝突がそれより近くで顕在化すると計画失敗 → `_apply_emergency_stop` の **ego_max_accel×2.0（ハードコード、`src/simulation/integrated_simulator.py:732`）** で slam する。② 横から進入する歩行者は「同時刻衝突」が直前まで顕在化しない。
   - 対応案（組み合わせ推奨、要実験）:
     - (i) **専用ブレーキ候補の追加**: Ti×tv 格子の外に、一定減速度（例 −1.5/−2.0/−2.5 m/s²）で停止する短時間プロファイルを数本だけ常時追加（候補爆発なしで「3〜8m で確実に止まる」選択肢を埋める）。
     - (ii) **緊急停止減速度の config キー化**: `_apply_emergency_stop` の ×2.0 を新キー（例 `ego_emergency_decel`）に。緩めると制動距離が伸びるので衝突率の検証必須。
     - (iii) **速度依存の予防トリガ**: 現在の固定 clearance トリガを RSS 風に `trigger = v·τ + d0`（例 τ=0.8s, d0=1.0m）へ。接近速度が高いほど早く CAUTION に落ち、低速ではうるさくない。validate のヒステリシス検証（recover > trigger）を速度依存にする設計に注意。
     - (iv) フォールバック: `min_t` を 2.0s に下げる（Ti 格子が dt=0.1 刻みなので候補数 ~2.8 倍 → 計画時間増。Ti 刻みの粗格子化と併せて検討）。
   - **守るべき不変条件**: 衝突判定は同時刻のみ / 停止プロファイル必含（tv 格子は 0 まで下方拡張済み、上側に超過候補なし）/ 曲率は運動学的制約で緩和禁止。
3. **減速度のシナリオ間統一**: 現状 `ego_max_accel` が S1/S3=2.0、**S2=1.5**（緊急減速 −4.0 vs −3.0 の不統一）。S2 を 2.0 に揃える（または (ii) の新キーで明示的に統一）。
   - 注意: 状態機械の予防パラメータは現在**意図的に**シナリオ別（S1/S3: trigger 3.0 / recover 3.5 / caution speed 0.6x、S2: 1.5 / 2.0 / 0.8x）。S2 に強い予防減速を入れると対向流の中に滞留して**衝突が発生する**ことを実験で確認済み（YAML コメントにも記載）。ユーザーの「統一」が縦減速度のみを指すのか、状態機械パラメータも含むのかは**作業前に確認**すること。(iii) の速度依存トリガが入れば S2 の特例は不要になる可能性が高い（S2 は低速接近なのでトリガが自然に小さくなる）。

## 4. アーキテクチャの押さえどころ（今回触る箇所）

- 候補生成: `frenet_planner.py` `_generate_frenet_paths`（tv 格子 ~L365、Ti 格子はその直上。停止候補必含）。チェック: `_check_paths`（speed/accel → `_curvature_feasible`（低速レジーム含む）→ lat_accel → road_bound → collision の順、全て index 0 スキップ）。棄却内訳は `planner.last_check_stats`、シミュレータが `metrics['n_collision_rejected']` に記録。
- 状態機械: `state_machine.py`。昇格 = 計画失敗 or 予防トリガ（clearance < trigger、failures は増えない）。復帰 = clearance キー優先・なければレガシー中心間距離から導出。CAUTION で target_speed×speed_mult。曲率は絶対に緩和しない（EMERGENCY は accel×3 と lat_accel×2 のみ）。
- 緊急停止: `integrated_simulator.py` `_apply_emergency_stop`（×2.0 ハードコード、位置は慣性で前進継続）。
- 設定: `SimulationConfig` dataclass が唯一の情報源。**キー追加は dataclass + validate_config + save_config の3点セット**。
- 派生 YAML: `scenario_0X_{cv,lstm}.yaml` は幾何・状態機械パラメータを base と**共有**する。**base を変えたら派生6本も必ず同じ変更**（prediction_method / sgan_model_path / output_path だけが差分）。
