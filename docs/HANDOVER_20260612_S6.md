## 2. 論文修正の方針（ユーザー決定事項）

1. **評価は単円（ego_radius 1.0 m）構成に絞る**。multi_circle / footprint 整合計画は**将来課題**として limitations に1段落（根拠: §1-6 の「カバー構成依存の非単調性」= シナリオ設計が単円前提で、車両形状整合な計画とシナリオ再設計は独立した研究課題）。
2. **「時空間（同時刻）衝突判定」は維持**（ユーザー提案の「予測軌道全体での判定=時間平坦化」は検討の上却下）。却下根拠は論文の設計判断記述や査読対応に使える: ①凍結ロボット問題（S1 は歩行者14人、うち停止時前方に約12人 — 5s ホライゾン×1.4m/s の掃引和が走路を恒久閉塞 → 実験的にも stall→timeout を実証済み）②「保守化⇒安全」が閉ループで不成立（第4世代 circle で inf1.50 が S1 1/20・S3 3/20 衝突 — 過剰減速による滞留が後続歩行者との遭遇を増やす。**注意: この衝突データは第4世代。第5世代の inflation sweep は全衝突ゼロ**）③予測誤差へのロバスト化は分布対応（robust）が正道で、★1 がその定量根拠。
3. **安全寄り構成の推奨 = robust(ε=0)**: §1-1 の「S2/S3 両軸支配・S1 も Time 同等で MinDist 改善」を根拠に、「安全第一なら robust eps0（+必要に応じ inflation 1.1）」を margin_control 実験の結論として主張可能。
4. 中期の将来課題として**時間バッファ付き同時刻判定**（t±Δ 窓、Δ→∞ が平坦化・Δ=0 が現行の連続スペクトル）に言及可（Gen-5c 候補）。

## 4. 手法節に書くべき第5世代アーキテクチャ（チューニング値）

第4世代からの差分（コミット 0251f41〜1ee1a6f）。全て単円構成の論文評価に効いている:

- **全域速度サンプリング + ブレーキ候補ラダー**: 縦方向候補に Ti=0.5〜min_t（0.5s 刻み）の quartic 停止候補を常時追加（横は現在 d ホールド）、計画ホライゾン max_t=5.0s まで静止サンプルでパディング（同時刻衝突判定の健全性維持）。
- **安全速度エンベロープ（全状態）**: target ≤ √(2·a_env·(clearance_ahead − standoff))。clearance_ahead = 前方半平面の歩行者への surface-to-surface 距離（braking は前方衝突にのみ有効、横/後方で速度を拘束しない）。v_env=0 のとき**停止指令 max_stop_distance**（= clearance_ahead − 0.2）を発行し「残距離内で停止する候補のみ可」（受動ホライゾンの停止先送りを遮断）。
- **適応緊急減速**: 計画失敗時、必要減速 = v²/(2·max(clearance_ahead−0.2, 0.05)) を [ego_max_accel, ego_emergency_decel] に clip。**clearance_ahead 非有限（横からの収束・静的障害物）なら最大レート**にフォールバック。
- **予防 CAUTION トリガ（RSS 風速度依存）**: clearance < trigger + τ·v で NORMAL→CAUTION。**回復は速度対応**: clearance > max(recover_gate, trigger + τ·v)（チャタリング帯の根治）。
- **パラメータ（YAML が正、変更禁止）**: 全シナリオ共通 ego_emergency_decel 4.0 / agent_radius 0.3 / ped_radius 0.2 / ego_radius 1.0 / ego_max_accel 2.0 / ego_max_curvature 0.2。S1: エンベロープ a2.0/standoff0.3・トリガ 1.0+0.25v・recover 2.0/2.0・caution_speed_mult 0.6。S3: a1.2/standoff1.0・トリガ 1.0+0.8v・recover 3.5/3.5・caution_speed_mult 0.6。**S2: エンベロープ無効**（強い予防減速は対向流内滞留→衝突を悪化、実験済み — 論文では「正面交錯シナリオでは予防減速を無効化」とパラメータ表に明記すれば足りる）、トリガ 1.5（headway なし）・recover 2.0/2.0・caution_speed_mult 0.8。
