## 主結果

### 1. 予測精度（ADE）は全手法・全シナリオで3〜10倍改善

| campaign | CV | LSTM | SGAN |
|---|---|---|---|
| comfort_s1 | 2.397→**0.438** | 2.280→**0.616** | 2.378→**1.090** |
| comfort_s2 | 2.326→**0.282** | 2.012→**0.312** | 2.113→**0.588** |
| comfort_s3 | 2.474→**0.243** | 2.252→**0.296** | 2.129→**0.680** |
| rand_s1 | 2.319→**0.398** | 2.157→**0.617** | 2.260→**1.054** |
| rand_s2 | 2.199→**0.271** | 1.891→**0.297** | 1.987→**0.578** |
| rand_s3 | 2.238→**0.231** | 2.022→**0.305** | 2.052→**0.638** |

修正前の ADE 2.0〜2.5 はバグ起因の人工的な悪化だった。

### 2. 手法間ランキングが反転: CV > LSTM > SGAN（CV が全シナリオで最良、有意）

修正前は3手法ほぼ横並び（むしろ LSTM/SGAN がわずかに優位）だったが、修正後は全キャンペーンで CV が最良。ランダム化 GT での Welch 検定（n=20/手法）:

- rand_s1: CV 0.398 vs SGAN 1.054 (p=2.0e-27)、vs LSTM 0.617 (p=8.5e-15)
- rand_s2: CV 0.271 vs SGAN 0.578 (p=4.6e-13)、vs LSTM 0.297 (p=4.0e-03)
- rand_s3: CV 0.231 vs SGAN 0.638 (p=1.2e-12)、vs LSTM 0.305 (p=1.8e-08)

解釈: 本シミュレーションの地上真値は Social Force Model でほぼ滑らかな等速歩行のため、観測が正しければ CV 外挿が最強になるのは整合的。SGAN のサンプリングノイズ（best-of-20 でも）が純粋に不利に働く。**「深層予測がCVに勝つ」前提の論文記述は全面見直しが必要。** 旧 ★2 の「GT 分布化で S3 の SGAN ADE 優位が消失」という知見は前提自体が消滅（SGAN 優位はバグの産物で、最初から存在しなかった）。

### 3. ★2 の他の結論の存否

- **S2 のランダム化 GT 衝突**: 31/60 → **24/60**。定性的結論（決定論 GT の衝突ゼロは天井であり、GT 分布化で手法非依存の graze が出る）は**維持**。手法別 mean collision_count は CV 0.30 / LSTM 0.50 / SGAN 0.40。
- **KDE-NLL の「重度の過信（16〜19 nats、クリップ上限近傍）」は大部分がバグの産物**: 修正後は LSTM 1.8〜3.0 nats、SGAN 5.4〜11.8 nats。LSTM は妥当な較正域に入り、SGAN は依然高め（特に S1 の 11.8）。★5（conformal 較正）の動機付けは「全モデル重度過信」から「SGAN の分布が GT に対して広い/ずれる」に書き換えが必要。
- **衝突ゼロの天井（決定論 GT）**: 維持（comfort_s* 全手法 0 件）。

### 4. 挙動面の変化

- S3 で LSTM/SGAN の走行時間が大幅増（19.3→27.0s / 18.5→26.0s）。予測が正確になり深層モデル側がより慎重な回避をとる。
- comfort_s1 の CV min_dist が 3.57→1.78 に減少（正確な予測で必要十分なマージンに最適化される方向）。min_dist の絶対値は手法・シナリオ間で増減が混在。
- rms_jerk は全体に微改善。

## 留意点

- ADE/FDE/NLL の絶対値には、レビュー M-1/M-2 の**時刻アンカーずれ（最大0.3s、未修正）**が残っており、標準 SGAN 評価プロトコルとの厳密な互換はまだない。手法間比較は同条件なので有効。
- 旧 `output/comfort_s*` / `output/benchmark_rand_s*` は比較用に残置。**論文表は `*_obsfix` 側から生成すること**（latex_table.txt は各 obsfix ディレクトリに生成済み）。
- `output/benchmark_nll_s*` は comfort_s* とビット一致の遺物につき再計測不要（obsfix 系が NLL 列を含む）。
- exp_footprint / exp_margin_control（★1）/ exp_sm_clearance / exp_proc_planning の各実験キャンペーンも同バグの影響下で取得されたものであり、結論を引用する場合は再計測の要否を判断すること（特に ★1 の chance-constrained 比較は予測分布の形が変わるため要再実行）。
